🚀 はじめての方へ

初回セットアップから初めてのシフト発表まで 30 分。手順は 5 ステップです。

  1. 初回セットアップ — トップページの「14日間 無料で試す」から登録、または 管理画面 で初回オーナーパスワードを設定。
  2. 店舗設定 — 設定タブで店舗名・週予算・営業セッション（ランチ/ディナー等）・必要人数マトリクスを入力。
  3. スタッフ登録 — スタッフタブで CSV 取込 or 手動追加。本職・時給・週時間・固定休を設定。
  4. 希望収集 — 「🔗 全員のリンク」ボタン → クリップボードにコピー → LINE グループに貼付。スタッフはスマホで希望入力。
  5. AI 生成 → 確定 → 通知 — シフト編成タブで「🤖 AI 自動生成」→ 結果確認 → 「✓ 確定する」→ LINE 通知文を一斉送信。
🎮 まずは触ってみたい方へ: /demo でログイン不要のデモ環境がご利用できます（10名スタッフ・サンプルシフト入り）。

👥 スタッフ管理

CSV で一括登録するには？
スタッフタブ → 「📥 CSV取込」ボタン。各列の意味:
名前,本職ID,時給,週最低,週最大,固定休(0-6),スキル(1-5)
例: 山田太郎,hall,1100,10,28,0 6,3（日曜と土曜が固定休）
兼任ポジションの設定方法
スタッフ編集モーダル → 「兼任可能ポジション」セクションでチェック。AI 生成時、本職に空きがあれば本職優先、無ければ兼任で配置されます。
スキル値の使い方
1〜5 でスキルレベルを設定。AI スコアに「スキル」要素として反映されます（既定重み 10%）。新人=1、ベテラン=5。

📝 希望収集

希望提出リンクの配り方
希望収集タブの「💬 LINE用 全員リンク生成」 → クリップボードにコピー → LINE グループに貼付。各スタッフは自分の URL をタップして希望入力。
「希望」「必須」「不可」の違い
  • 希望: 入れたら入りたい（スコア +85〜100）
  • 必須: 絶対入りたい（スコア +100、AI が最優先で入れる）
  • 不可: 入りたくない（スコア 0、AI は避ける）
未提出スタッフの催促
ダッシュボードに未提出スタッフが赤バッジで表示されます。「全員リンク生成」を再実行して再送信できます。

📅 シフト編成

AI 生成の判断基準は？
7 つのハード制約（必ず守る）と 5 つのソフト制約（スコアで最適化）で生成しています。詳細は アルゴリズム仕様書。 自動テストで違反ゼロを保証: テストページ。
スタッフを手動で入れ替えたい
シフト編成タブで、入れ替えたいスタッフをドラッグ → もう一方にドロップ。同ポジション間は即入替、異ポジション間は確認ダイアログが出ます。
急な欠勤対応（代打推薦）
該当アサインをクリック → 「🆘 欠勤想定 → 代打推薦」ボタン → AI が条件を満たすスタッフ 5 名をスコア順に表示 → 「入替」をクリックで完了。
「先週からコピー」「テンプレ保存」の使い方
先週からコピー: 前週のシフトをそのまま今週に複製（曜日合わせ）。
テンプレ保存: 「通常週」「夏休みシフト」等の名前で保存、後で別の週に適用可能。

✅ シフト確定・通知

確定するとどうなる？
・週の状態が「下書き」→「確定済」に変わります
・スタッフがポータル URL を開くと、希望入力UIではなく確定シフトの閲覧UIに切り替わります
・LINE 通知文が自動生成されるので、コピーしてLINEに貼り付けて配信できます
確定後に変更したい場合
「下書きに戻す」ボタンで一度下書き状態に戻し、編集後再確定。スタッフは閲覧モードから希望入力モードに一時戻りますが、データは保持されます。

📱 スタッフポータル

スタッフ側の使い方
店長から LINE で送られてきた URL をスマホで開く → そのまま希望入力 or 確定シフト確認。アプリインストール・登録不要。
URLを失くしたら？
店長に連絡 → 管理画面で「🔗 リンク」ボタンを再実行で同じ URL が再表示されます（トークンは保存済み）。

⚙️ 詳細設定

必要人数マトリクスとは？
曜日 × 時間帯 × ポジション の表で「何人必要か」を定義。AI はこのマトリクスを満たすようにシフトを組みます。
労務ルールの設定
設定タブ「6. 労務ルール」で週/日上限・連勤・最低休日を設定。AI 生成時に絶対守られます（ハード制約）。
アルゴリズム重み調整
設定タブ「7. アルゴリズム重み調整」で 5 要素（希望充足・ポジション・公平性・コスト・スキル）の重みを店舗ごとに調整可能。スライダーで 0〜100 の値を変更し、自動正規化されます。
バックアップ・復元
設定タブ「8. データバックアップ」→「📥 JSONでダウンロード」で全データをエクスポート。「📤 JSONから復元」で別環境への移行・障害復旧に使えます。週次のダウンロード推奨。

🛠 トラブルシューティング

画面のスタイルが崩れている / レイアウトが変
ブラウザのキャッシュが古い可能性があります。
1. シークレットウィンドウ (Ctrl+Shift+N) で開く → これで動けばキャッシュ問題確定
2. 通常ウィンドウなら DevTools (F12) → Application → Storage → 「Clear site data」 → リロード
AI 生成のカバー率が低い
考えられる原因:
・必要人数 > スタッフキャパシティ（労務ルール上限で詰まる）
・固定休日が重なって候補不足
・希望が「不可」だらけで配置先がない
対処: 設定タブで必要人数マトリクスを下方修正、またはスタッフ追加。
スタッフが希望URLを開けない
・URL がコピー切れていないか確認（最後まで全文選択でコピー）
・「リンク」ボタンを再実行 → 既存と同じ URL が再生成されます
・トークン削除を実行した場合は新規発行が必要
パスワードを忘れた
メールで support@in-dx.jp までご連絡ください。本人確認の上、サーバ側でリセットさせていただきます。

❓ よくある質問

14日無料トライアル後、自動で課金されますか？

いいえ、自動課金されません。クレカ登録不要でお試しいただけます。継続される場合のみ別途お手続きください。

複数店舗運営は対応してますか？

現在は 1 店舗 1 アカウント。多店舗対応は 2026 年内のロードマップに入っています。先行ご相談で個別 Cloud Run インスタンス発行でご提供可能です。

既存のシフト管理ツール（Air シフト等）からのデータ移行は？

スタッフ情報は CSV 取込で一括登録できます。シフトの履歴データはお問合せください、移行スクリプト個別対応します。

VOIXA との違いは？

VOIXA = 予約電話の自動応答 AI、Shifty = シフト作成の自動化 AI。両方契約で月¥1,500 OFF。

データの安全性は？

Google Cloud Platform 上で運用、全通信 HTTPS、データは Firestore で暗号化保管、パスワードは bcrypt ハッシュ化、CSP 等のセキュリティヘッダ実装済み。バックアップもオーナー側でいつでも JSON 取得可能。

解約方法

解約は support@in-dx.jp までメールにてお申し出ください。24 時間以内に解約処理いたします。違約金等は発生しません。解約後 30 日間データ保持、再開時に復元可能。
